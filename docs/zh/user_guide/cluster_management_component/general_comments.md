# 概述

集群管理组件，主要服务于集群场景，根据应用场景，分为以下三个子组件。

**表 1**  组件分类

|组件名称|应用场景|能力描述|
|--|--|--|
|Controller|集群运行时的运维管理场景。|控制器，完成集群内所有Server的业务状态管控、PD身份管理与决策、资源管理决策等，是整个集群的状态管控器和决策大脑，详情请参见[控制器（Controller）](./controller.md)。|
|Coordinator|集群运行时的推理请求入口场景。|调度器，是用户推理请求的入口，接收高并发的推理请求，进行请求调度、请求管理、请求转发等，是整个集群的数据请求入口，详情请参见[调度器（Coordinator）](./coordinator.md)。|

<br>

**特性支持度**

|特性|Atlas 300I Duo 推理卡+Atlas 800 推理服务器（型号 3000）|Atlas 800I A2 推理服务器|Atlas 800I A3 超节点服务器|
|--|--|--|--|
|单机服务部署|支持|支持|支持|
|单机PD分离服务部署|不支持|支持|支持|
